You want to contribute to Checklist+?

Here are a few things to know:
- **Important Stylistic Note** -- my intent is to keep this simple and focussed on rapid, intuitive note-taking, time-management, and task-tracking. Arguably it's already more complicated than it should be... so... I will probably reject changes that add noise.
- **Important Procedural Note** is to know there are two main branches. `main` is the "live" stable branch which pushes code into thousands of people's editors. `dev` is the place where development happens.
  - When beginning work on a patch, please start your branches from the latest commit on the `dev` branch. If you've already made some commits, please `git rebase develop` and then continue.
  - When making a pull request, please target it back to the `dev` branch. Pull requests targetting `master` will be rejected.
- Everything else is pretty standard:
  - Trim trailing spaces.
  - Unix line endings.
  - Spaces, not tabs.
  - Run your code through PEP8, or some linting program.
